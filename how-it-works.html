HOW CLAUDE SKILLS WORK

A structured framework to extend Claude's reasoning and execution capabilities.

CORE CONCEPT

01

Skills Are Modules

Each skill is a self-contained capability — a folder with metadata and instructions that teach Claude new behaviors.

02

Triggered by Commands

Users invoke skills with slash commands like /setup-ghostclaw or natural language that matches triggers.

03

Context-Aware Execution

Skills have access to conversation history, file system, git, APIs, and can chain together for complex workflows.

SYSTEM ARCHITECTURE

GhostClaw System Flow
// Message received (WhatsApp/Telegram)
orchestrator.route(message)
→ check for skill trigger
→ load skill prompt.md
→ spawn agent with skill context
→ agent executes task
→ return result to user

AVAILABLE TOOLS

Skills can use any Claude Agent SDK tools:

File System

  • Read/Write files
  • Search codebase (Grep, Glob)
  • Edit existing files

Git Operations

  • Commit changes
  • Push to remote
  • Create branches

Bash Commands

  • Run any shell command
  • npm/pip/cargo operations
  • Deploy to Vercel/Railway

Web Access

  • Fetch URLs
  • Search the web
  • API calls

Task Management

  • Schedule tasks (cron)
  • Background loops (Ralph)
  • Multi-agent teams

Messaging

  • Send messages immediately
  • Register new groups
  • Voice transcription/TTS

REAL-WORLD EXAMPLES

/add-morning-briefing

What it does: Creates a scheduled task that runs every morning, checks your calendar/email/news, and sends a summary.

Tools used: Schedule task, fetch APIs, send message

/run-ralph

What it does: Autonomous loop that reads a markdown checklist, completes each task in a fresh context, commits after each step.

Tools used: File system, git, bash, task scheduling

/variant-pipeline

What it does: Takes Variant.com design exports, applies to codebase, generates TikTok scripts, builds Remotion videos.

Tools used: File system, git, bash, web fetch
