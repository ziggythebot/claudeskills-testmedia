Getting Started with GhostClaw

Installation

git clone https://github.com/your/ghostclaw.git cd ghostclaw npm install npm run dev

First-Time Setup

Run the setup skill to build your AI personality:

/setup-ghostclaw

This guides you through:

  • WhatsApp/Telegram authentication
  • Personality building (soul generation)
  • Channel registration
  • Service configuration

Add Your First Skill

Try voice transcription:

/add-voice-transcription

You'll need an ElevenLabs API key. The skill will guide you through setup.

Test It Out

Send a voice note to your WhatsApp/Telegram bot. It should transcribe automatically.

Creating Your First Skill

Skill Structure

Skills are folders with two files:

.claude/skills/my-skill/ ├── skill.json # Metadata └── prompt.md # Instructions for Claude

1. Create skill.json

{ "name": "my-skill", "description": "What this skill does", "triggers": ["my-skill", "trigger phrase"], "version": "1.0.0", "author": "You", "tags": ["category"] }

2. Write prompt.md

This teaches Claude what to do:

# /my-skill - Short Description ## What This Does Clear explanation of the skill's purpose. ## Workflow Step 1: First action Step 2: Second action Step 3: Result ## Example User: /my-skill Agent: [Does the thing]

3. Test It

Run the skill in your chat:

/my-skill

Best Practices

  • Keep prompt.md under 500 lines
  • Use clear section headers
  • Include examples
  • Document required API keys
  • Test edge cases

Autonomous Workflows with Ralph

What is Ralph?

Ralph is a task loop that runs overnight, completing checklists autonomously. Each task runs in a fresh context window with commits after each step.

Create a Task File

Make a markdown file with checkboxes:

# Project Tasks - [ ] Create user model - [ ] Add login endpoint - [ ] Add registration endpoint - [ ] Write tests - [ ] Deploy to staging

Start Ralph

/run-ralph tasks.md

How It Works

  1. Reads task file, finds first unchecked task
  2. Spawns fresh agent with that task
  3. Agent completes task and commits
  4. Marks task as done in file
  5. Messages progress to you
  6. Repeats until all tasks done

Best Practices

  • Break tasks into small, testable chunks
  • Put tests after implementation tasks
  • Monitor progress messages
  • Review commits in the morning
  • Use /stop-ralph to halt if needed

When to Use Ralph

  • Multi-step feature implementation
  • Refactoring large codebases
  • Running test suites overnight
  • Data migration scripts
  • Documentation generation
